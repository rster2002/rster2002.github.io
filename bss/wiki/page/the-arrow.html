Using the arrow

You may have seen '=>' in for example the 'if' command. This if for closing the compiled 'execute' command. If we have the following example:

Bluestone Script

{{ if i > 10 => }} say hi

We can see that at the end of the command there is an arrow. This indicates to the compiler it should look at the rest of the line like a new command.

If we wanted to extend the compiled 'execute' command, we would not add a arrow at the end. Like here:

Bluestone Script

{{ if i > 10 }} at @s if block ~ ~-1 ~ minecraft:stone run say stone
