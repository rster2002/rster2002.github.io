Routing

Because it makes little to no sense

Read this before you start

This document might be unstructured and chaotic and does not include everything you need to know, but will probably get you back on track when you're stuck. Keep an eye out for notes like this one because it may just contain the information you need to continue with your work. You might need to also use your own ip addresses to configure certain things, so be mindful of that. Also note that there might be some errors in this document.

You might need to reset configuration if you've done something wrong. If this is the case, add no in front of the command to remove it from the configuration. For example if you've set the wrong ip address using: ip address 192.168.10.1 255.255.255.0, you can undo your config by writing a new command: no ip address 192.168.10.1 255.255.255.0.

RIPv2

Hardware layout

These are the routers and switches in this network.

Set unmatched route (default route)

The unmatched (or default) route is the route that will be taken if there's no match within the network. In this case it's at router 1:

xxxxxxxxxx
R1(config)#ip route 0.0.0.0 0.0.0.0 G0/2
%Default route without gateway, if not a point-to-point interface, may impact performance
xxxxxxxxxx
ip route 0.0.0.0 0.0.0.0 <outbound interface>

Note

The default route should only be set at the router at the edge of the network. In this case it's R1 because it's connected to the rest of the internet.

Configure RIP

This example will look at R1, but the configuration for the other routers are similar (view notes below).

xxxxxxxxxx
R1(config)#router rip
R1(config-router)#version 2
R1(config-router)#no auto-summary
R1(config-router)#network 192.168.1.0
R1(config-router)#network 192.168.2.0
R1(config-router)#passive-interface G0/1
R1(config-router)#default-information originate
xxxxxxxxxx
router rip
version 2
no auto-summary
network <connected networks...>
passive-interface <passive interfaces>
default-information originate
Note

Some important things to consider when configuring RIP:

  • network <connected network>: The connected networks are all the networks that are directly connected to this router. So in the same of R1, it's the network between S1 and the network between R2.
  • passive-interface <passive interface>: The passive interfaces should be the interface that should not be included in the RIP process. In the case of R1, it's G0/1 because that's the interface connected to the switch. If it were a router, the interface would not be set to passive.
  • default-information originate: This is only for the router that's the endpoint of the network and connects to a WAN (internet).

You can use no to remove configuration. So if I mistakenly configured network 192.168.11.0 even though it's not directly connected, type: no network 192.168.11.0. Usually just putting no in front of the command you've run by accident will revert the changes of that command.

Also remember to configure the interfaces with an ip-address and changing their state to up.

Working with underlying VLAN's

Remember to add the underlying VLAN's to the RIP configuration. So if we have the VLAN's shown in InterVLAN, remember to go into the router rip configuration and add the networks:

xxxxxxxxxx
R1(config)#router rip
R1(config-router)#network 192.168.10.0
R1(config-router)#network 192.168.20.0
R1(config-router)#network 192.168.30.0

Not doing this will prevent devices from talking to devices within the VLAN's and this includes DHCP servers. So when working with a DHCP server and a relay, remember to add it to the routers config.

Set TRUNK

Then set the out interface to be the trunk (G0/1). The interface is the interface going to the router.

xxxxxxxxxx
Switch(config)#int G0/1
Switch(config-if)#switchport mode trunk
Switch(config-if)#switchport trunk native vlan 88
Switch(config-if)#switchport trunk allowed vlan 10,20,30
Switch(config-if)#exit

Configure relay

Configure the relay by adding a helper ip-address to the inner interface. So in the following scenario where R1 is the DHCP server and R2 is an underlying router, the G0/1 interface of R2 should be configured. The IP address should be the address of the interface of R1 for that section of the network. So if R2 also had another router connected it should also use the IP address of R1. So in this case it would be the address of the G0/0 interface of R1 (ip address 192.168.2.1, check RIPv2)

xxxxxxxxxx
R2(config)#int g0/1
R2(config-subif)#ip helper-address 192.168.2.1
R2(config-subif)#exit
xxxxxxxxxx
int <inner interface>
ip helper-address <helper ip-address>
Note when working with VLAN's

Adding the helper address to the interface itself won't cut it. You'll need to add it to the sub-interfaces as well:

xxxxxxxxxx
R3(config)#int g0/2.10
R3(config-subif)#ip helper-address 192.168.2.1
R3(config-subif)#exit
R3(config)#int g0/2.20
R3(config-subif)#ip helper-address 192.168.2.1
R3(config-subif)#exit
R3(config)#int g0/2.30
R3(config-subif)#ip helper-address 192.168.2.1
R3(config-subif)#exit
R3(config)#int g0/2.88
R3(config-subif)#ip helper-address 192.168.2.1
R3(config-subif)#exit

You only need to do this of course if the DHCP server is not the trunk router. Otherwise you don't have to do this.
